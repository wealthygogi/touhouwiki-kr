TH12 Content Standardization Design

Goal: Standardize characters and dialogue files for TH12 (Touhou Seirensen) according to the content-standardization-guide.md.

1. Dialogue Standardization (docs/mainline/gen2/th12/dialogue/*.mdx)​

Component Mapping​

  • Convert legacy tables | Speaker | Text | to <Dialogue> components.
  • Convert action rows | | Action | to <Action> components.
  • BGM markers ♪ [Title](link) should be treated as <Action> or kept as is within the dialogue flow if they were in the table. Usually, they are in action rows.

Speaker Mapping​

  • Left (Player): 레이무, 마리사, 사나에
  • Right (Boss): 나즈린, 코가사, 타타라 코가사, 이치린, 쿠모이 이치린, 무라사, 무라사 미나미츠, 쇼, 토라마루 쇼, 뱌쿠렌, 히지리 뱌쿠렌, 누에, 호쥬 누에
  • ??? Mapping:
    • Stage 1: 나즈린
    • Stage 2: 코가사
    • Stage 3: 이치린
    • Stage 4: 무라사
    • Stage 5: 쇼
    • Stage 6: 뱌쿠렌
    • Extra: 누에

Attributes​

  • comment: Extract text in brackets (e.g., (어라?)) from the end of the line into the comment attribute.

2. Character Profile Standardization (docs/mainline/gen2/th12/characters/*.md)​

Format​

  • Use <CharacterProfile> component.
  • Attributes: nameKr, nameJp, nameEn, image, title, species, ability.
  • Critical: Every line in the body must have (trailing two spaces).
  • Critical: Remove leading indentation (full-width spaces 　).

3. Overview Files (characters/overview.mdx, dialogue/overview.mdx)​

  • Remove slug from frontmatter.
  • Simplify <TileList>:
    • Keep only title and link.
    • Remove description, itemCount, etc.

4. Verification​

  • Run npm run build or yarn build to check for broken links or MDX errors.
  • Use lsp_diagnostics to check for syntax errors.
