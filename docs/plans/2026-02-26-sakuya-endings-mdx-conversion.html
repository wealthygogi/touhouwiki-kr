Design Doc: th07 Sakuya Endings MDX Conversion

Goal​

Convert docs/mainline/gen1/th07/dialogue/sakuya-endings.md to .mdx format while adhering to specific precision rules for custom components and tag parsing.

Scope​

  • Target: docs/mainline/gen1/th07/dialogue/sakuya-endings.mdx
  • Source: docs/mainline/gen1/th07/dialogue/sakuya-endings.md
  • Components: Dialogue, Action from @site/src/components/Dialogue.

Transformation Rules​

1. Metadata​

  • sidebar_position: Set to 9.
  • Title and description remain the same.

2. Imports​

Add import { Dialogue, Action } from '@site/src/components/Dialogue'; at the top of the file (after frontmatter).

3. Dialogue Parsing​

  • Pattern: <tl$Speaker:> -> <Dialogue speaker="Speaker">
  • Attributes: Use double quotes (e.g., speaker="사쿠야").
  • Multi-line dialogue: The source uses <l$> to indicate line breaks. These will be removed as the text is consolidated into the <Dialogue> component.

4. Action Parsing​

  • Any narrative text without a speaker tag will be wrapped in <Action>.
  • Tags like <c$ENDING No. 8$> will be converted to <Action>ENDING No. 8</Action>.
  • BGM (if found) will be standardized to <Action>♪ Title</Action>.

5. Comments in Parentheses​

  • If a dialogue line contains text in parentheses, that text will be moved to the comment prop of the <Dialogue> component.
  • Example: <Dialogue speaker="사쿠야" comment="미신이었나......" />

6. File Cleanup​

  • The original .md file will be deleted after the .mdx file is successfully created and verified.

Verification Plan​

  1. Text Preservation: Manually compare the generated MDX with the original MD to ensure no dialogue or narrative text is missing.
  2. LSP Check: Run lsp_diagnostics on the new .mdx file to ensure no syntax errors.
  3. Visual Consistency: Ensure the structure matches sakuya-scenario.mdx.
